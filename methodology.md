# 麦肯锡核心方法论

> 麦肯锡（McKinsey & Company）核心方法论整理

---

## 📚 一、麦肯锡七步问题解决法（7-Step Problem Solving）

麦肯锡经典问题解决框架，适用于复杂商业问题的结构化分析：

| 步骤 | 名称 | 说明 |
|------|------|------|
| 1 | **定义问题**（Define the Problem） | 明确真正要解决的问题，避免"解决错误的问题" |
| 2 | **分解问题**（Break the Problem into Components） | 使用问题树（Issue Tree）将大问题拆解为可管理的小问题 |
| 3 | **优先级排序**（Prioritize the Components） | 按80/20原则，聚焦对结果影响最大的子问题 |
| 4 | **制定工作计划**（Develop a Workplan） | 为每个子问题分配资源、时间和分析方法 |
| 5 | **分析问题**（Analyze the Components） | 收集数据、进行定量分析、访谈验证 |
| 6 | **综合结论**（Synthesize the Findings） | 从分析结果中提炼核心洞察（So What?） |
| 7 | **呈现建议**（Present the Recommendations） | 用金字塔原理结构化呈现，讲故事的方式汇报 |

### MECE 原则

麦肯锡分析的核心原则：**M**utually **E**xclusive, **C**ollectively **E**xhaustive（相互独立，完全穷尽）

- **相互独立**：各子问题之间不重叠
- **完全穷尽**：所有子问题加总覆盖原问题

## 📈 二、金字塔原理（Pyramid Principle）

麦肯锡经典呈现方法，由芭芭拉·明托（Barbara Minto）提出：

```
        核心结论（Answer First）
           /    |    \
     支撑论点1  支撑论点2  支撑论点3
        /  \      /  \      /  \
     数据1 数据2 数据3 数据4 ...
```

**原则：**
1. **结论先行**（Answer First）：先说结论，再讲分析
2. **以上统下**：上层是下层的总结，下层是上层的支撑
3. **归类分组**：同组思想属于同一逻辑范畴
4. **逻辑递进**：同组思想按逻辑顺序（演绎/归纳）排列

## 📊 三、战略分析工具箱

### 3.1 内部分析

| 工具 | 用途 |
|------|------|
| **VRIO 框架** | 分析企业内部资源与能力的竞争优势 |
| **价值链分析**（Value Chain） | 识别企业各环节的增值活动 |
| **成本结构分析** | 理解企业成本驱动因素 |

### 3.2 外部分析

| 工具 | 用途 |
|------|------|
| **PESTEL 分析** | 宏观环境（政治/经济/社会/技术/环境/法律） |
| **波特五力模型**（Porter's Five Forces） | 行业竞争格局分析 |
| **客户细分**（Customer Segmentation） | 按需求/行为对客户分组 |
| **竞争对手分析**（Competitor Mapping） | 定位竞争对手的战略位置 |

### 3.3 战略制定

| 工具 | 用途 |
|------|------|
| **BCG 矩阵** | 业务组合分析（明星/现金牛/问号/瘦狗） |
| **GE 矩阵**（McKinsey 9-Box Matrix） | 按市场吸引力和竞争力对业务进行九宫格定位 |
| **增长战略**（Ansoff Matrix） | 市场渗透/市场开发/产品开发/多元化 |
| **成本领先/差异化/聚焦** | 波特通用战略 |

## 📋 四、麦肯锡工作方法论

### 4.1 假设驱动（Hypothesis-Driven）

- 先提出假设，再收集数据验证
- 比"先分析再结论"更高效
- 核心问题：**"要验证这个假设，我需要什么数据？"**

### 4.2 以事实为基础（Fact-Based）

- 麦肯锡顾问不凭直觉说话
- 每个结论都要有数据支撑
- 区分**事实**（Fact）、**假设**（Hypothesis）、**观点**（Opinion）

### 4.3 80/20 原则（Pareto Principle）

- 80% 的结果来自20% 的原因
- 聚焦高影响力问题，不追求完美分析
- **"够好的决策，及时做出"** 优于"完美的决策，晚做出"

### 4.4 电梯测试（The Elevator Test）

- 能否在30秒内说清核心建议？
- 如果不能，说明逻辑还不够清晰
- 每次汇报前，逼自己用一句话总结

## 📁 五、麦肯锡经典文档结构

### 5.1 项目建议书（Project Proposal）

1. 背景与问题定义
2. 项目目标与范围
3. 方法论与工作计划
4. 团队与时间安排
5. 交付成果

### 5.2 中期汇报（Mid-Point Review）

1. 已完成工作概述
2. 初步发现与假设验证状态
3. 下一步工作计划（如有调整）
4. 客户反馈与确认

### 5.3 最终报告（Final Presentation）

1. **执行摘要**（Executive Summary）—— 一页纸说清所有核心建议
2. 问题定义与分析方法
3. 核心发现（Key Findings）
4. 战略建议（Recommendations）
5. 实施路线图（Implementation Roadmap）
6. 附录（详细分析）
